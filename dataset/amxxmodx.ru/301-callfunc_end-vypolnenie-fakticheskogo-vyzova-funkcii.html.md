# callfunc_end Выполнение фактического вызова функции.
[Ядро AMXx](http://amxxmodx.ru/core/) [amxmodx.inc](http://amxxmodx.ru/core/amxmodxinc/)
Кажется что вызывает функцию в другом плагине функция **callfunc_begin** , но на самом деле, фактически вызывает ее **callfunc_end** , то есть, другие функции из ряда **callfunc_*** формируют вызов функции, передают необходимые параметры, а **callfunc_end** уже непосредственно вызывает требуемую функцию.  
  
Инфо из **amxmodx.inc:**  
```
/* Make the actual call.  
* Return value of the function called. */  
native callfunc_end();
```
  
  
**Синтаксис:**  

callfunc_end()
  

    * Параметров нет  

Функция вернет значение выполнявшейся функции   
  
**Тип функции:**  
Native  
  
**Пример:**  
```
/* Plugin generated by AMXX-Studio */  
  
#include <amxmodx>  
#include <amxmisc>  
  
#define PLUGIN "[amxmodx.inc] callfunc_begin"  
#define VERSION "1.0"  
#define AUTHOR "Admin"  
  
  
public plugin_init() {  
    register_plugin(PLUGIN, VERSION, AUTHOR)  
      
    register_concmd("test_cmd","test_function")  
      
}  
public test_function(){  
      
    if(callfunc_begin("cmdWho","admincmd.amxx")==1)  
        callfunc_end()  
          
}  

```
  
  
**Описание:**  
Пример и описание точно такие же как для функции [**callfunc_begin**](http://amxxmodx.ru/core/amxmodxinc/300-callfunc_begin-funkciya-vyzyvaet-druguyu-funkciyu-po-ee-imeni-v-etom-ili-drugom-plagine.html), только теперь вы знаете правду, о том, кто и что вызывает.  
  
Это необходимо для дальнейшего изучения функций **callfunc_** , что бы не путаться в очередности вызовов.
