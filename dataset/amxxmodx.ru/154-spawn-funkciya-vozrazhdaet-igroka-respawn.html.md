# spawn Функция возраждает игрока (respawn)
[The Fun Module](http://amxxmodx.ru/fun/) [fun.inc](http://amxxmodx.ru/fun/funinc/)
Если у вас не классический сервер Counter-strike, а скажем Deathrun то там уже можно придумать плагины жизней и тому подобное, то есть нужно будет возродить игрока после его смерти. Функция **spawn** , как раз в этом вам и поможет.  
На официальном сайте написано что необходимо использовать функцию 2 раза подряд, что бы не было шибок с HUD обновлением, но это вызывает фантомное возрождение. По этому рекомендую использовать функции других модулей для респавна игрока.  
  
Инфо из **fun.inc:**  
```
/* Spawns entity. */  
native spawn(index);
```
  
  
**Синтаксис:**  

spawn ( index )
  

    * **index** - id Возрождаемого объекта  

  
  
**Тип функции:**  
Native  
  
**Пример:**  
```
/* Plugin generated by AMXX-Studio */  
  
#include <amxmodx>  
#include <amxmisc>  
#include <fun>  
  
#define PLUGIN "[fun.inc] spawn"  
#define VERSION "1.0"  
#define AUTHOR "Admin"  
  
public plugin_init() {  
    register_plugin(PLUGIN, VERSION, AUTHOR)  
    register_clcmd("say /spawn","user_spawn")  
  
}  
public user_spawn(id){  
    spawn(id)  
}
```
  
  
**Описание:**  
Мега плагин получился, возрождение по команде..  
Шутки в сторону, выполняя команду **say /spawn** , вызывается функция user_spawn(id). В теле которой указываем рассматриваемой функции id игрока,то есть id того кто выполнил команду. И игрока возрождает в точке появления его команды.
