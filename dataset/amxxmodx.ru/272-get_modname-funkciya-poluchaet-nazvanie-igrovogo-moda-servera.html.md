# get_modname Функция получает название игрового мода сервера
[Ядро AMXx](http://amxxmodx.ru/core/) [amxmodx.inc](http://amxxmodx.ru/core/amxmodxinc/)
Честно говоря я имел дело только с модом cstrike, по этому мне данная функция не была нужна, но видимо есть плиганы которые работают сразу на нескольких модах, но с какими то отличиями и для этого существует данная функция, что бы определить отличие модов.  
  
Инфо из **amxmodx.inc:**  
```
/* Returns a name of currently played mod. */  
native get_modname(name[],len);
```
  
  
**Синтаксис:**  

get_modname ( name[], len )
  

    * **name[]** - Массив для имени мода  

    * **len** - Максимальная длинна записываемого имени  

  
  
**Тип функции:**  
Native  
  
**Пример:**  
```
/* Plugin generated by AMXX-Studio */  
  
#include <amxmodx>  
#include <amxmisc>  
  
#define PLUGIN "[amxmodx.inc] get_modname"  
#define VERSION "1.0"  
#define AUTHOR "Admin"  
  
public plugin_init() {  
      
    register_plugin(PLUGIN, VERSION, AUTHOR)  
  
    new szModName[32]  
    get_modname(szModName,31)  
  
    server_print("Mod on server: %s",szModName)  
}
```
  
  
**Описание:**  
При запуске сервера или его перезагрузке, в функции инициализации плагина будет вызвана функция **get_modname** , которая запишет в заранее созданный массив имя запущенного мода и далее вывод в консоль сервера полученного названия.
