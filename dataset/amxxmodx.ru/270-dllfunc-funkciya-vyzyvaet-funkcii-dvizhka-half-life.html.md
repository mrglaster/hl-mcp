# dllfunc Функция вызывает функции движка Half-Life
[The Fakemeta Module](http://amxxmodx.ru/fakemeta/) [fakemeta.inc](http://amxxmodx.ru/fakemeta/fakemetainc/)
Мноиге из вас наверное заметили что заголовки у двух функций практически одинаковые ( dllfunc и engfunc) Но ни какой ошибки тут нету, обе функции вызывают функции движка, но они отличаются друг от друга, данная функция вызывает функции из набора **Used with dllfunc()**.  
  
Инфо из **fakemeta.inc:**  
```
native dllfunc(type,any:...);
```
  
  
**Синтаксис:**  

dllfunc ( type, [ ... ] )
  

    * **type** - Функция  

    * **[...]** - Параметры функции   

  
  
**Тип функции:**  
Native  
  
**Пример:**  
```
/* Plugin generated by AMXX-Studio */  
  
#include <amxmodx>  
#include <amxmisc>  
#include <fakemeta>  
  
#define PLUGIN "[fakemeta.inc] dllfunc"  
#define VERSION "1.0"  
#define AUTHOR "Admin"  
  
public plugin_init() {  
      
    register_plugin(PLUGIN, VERSION, AUTHOR)  
  
    register_clcmd("say /spawn","spawn")  
      
}  
public spawn(id){  
  
    dllfunc(DLLFunc_Spawn,id)  
}  

```
  
  
**Описание:**  
Данный пример не претендует на звание здравого кода, а набросан на скорую руку и можно считать его не правильным: попробуйте зайти в спектаторы и наблюдать за каким то конкретным игроком и выполнить команду say /spawn и вы увидите что что то пойдет не так, как если бы вы не за кем не наблюдая выполнили ту же команду, но привычно респавнились бы. Все таки модуль для работу с объектами а не игроками. На самом деле ни какой ошибки нету, думаю со временем вы сами поймете все.
