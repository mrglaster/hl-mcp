# plugin_pause Функция вызывается когда плагин переходит в состояние паузы
[Ядро AMXx](http://amxxmodx.ru/core/) [amxmodx.inc](http://amxxmodx.ru/core/amxmodxinc/)
Предположим плагин работает с базой данных и не смог соединиться с ней и переходит в состоянии паузы, а вы хотите записать какие то логи или выполнить еще какие то операции именно в этот момент. Функция **plugin_pause** вызывается именно в этот момент.  
  
Инфо из **amxmodx.inc:**  
```
/* Called when the plugin is paused. */  
forward plugin_pause();
```
  
  
**Синтаксис:**  

public plugin_pause ( )
  
  
**Тип функции:**  
Forward  
  
**Пример:**  
```
/* Plugin generated by AMXX-Studio */  
  
#include <amxmodx>  
#include <amxmisc>  
  
#define PLUGIN "[amxmodx.inc] pause"  
#define VERSION "1.0"  
#define AUTHOR "Admin"  
  
public plugin_init() {  
  
    register_plugin(PLUGIN, VERSION, AUTHOR)  
    pause("a")  
}  
public plugin_pause(){  
  
    server_print("Your plugin is translated in a pause state ")  
      
}
```
  
  
**Описание:**  
Ничего умнее я не смогу придумать, кроме как вывести в консоль сервера информацию о том что плагин перешел в состояние паузы.  
Я не встречал использование этой функции, так как все необходимые действия можно проделать просто до того момента как отправляется команда паузы.
